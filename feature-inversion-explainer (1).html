Feature Inversion · 特征反演

已知特征，
反推出图像

Mahendran & Vedaldi 论文里那句「by minimizing a feature reconstruction loss」到底是什么方法？拆开看，其实是一个只优化图像、不训练网络的循环。

“…invert features from convolutional networks by minimizing a feature reconstruction loss in order to understand the image information retained by different network layers…”
00 — 方向

先分清正向与反向

平时我们用网络都是「正向」：图片喂进去，得到特征。反演（invert）就是把箭头掉过来问一句——已知某层的特征，什么样的图像会产生它？能不能把图像还原回来？

Forward · 平时

图像 → 特征

x  →  VGG  →  φⱼ(x)

有现成的网络，一次前向就算出特征。简单。

Inverse · 反演

特征 → 图像 ？

φⱼ(x₀)  →  ？？？  →  图像

CNN 不可逆，没有现成的「逆函数」。于是改用优化去逼近。

为什么要费劲反演？因为反演结果能告诉你这一层到底保留了原图多少信息：能还原出清晰图 → 信息留得多；只能还原出模糊轮廓 → 这层把细节扔了。这正是 M&V 的研究目的——理解网络。

01 — 方法

「最小化特征重建损失」就是这个循环

既然没有逆函数，就把它当成一个优化问题来解：随机起一张图，反复修改它的像素，直到它的特征和目标对上。下面这张图就是整个过程——注意那条蓝色虚线回到的是「图像」，不是网络。

记下目标。把原图 x₀ 送进 VGG，取出第 j 层特征 φⱼ(x₀)。这一步只算一次，之后固定不动，当作要对齐的「靶子」。
随机起一张图。初始化一张纯噪声图 x（像素全随机）。它就是接下来要被反复打磨的对象。
算损失。把 x 也送进同一个 VGG，得到 φⱼ(x)，再和靶子比，差距就是特征重建损失（见下式）。
改像素。对损失求梯度，用梯度下降更新 x 的像素值，让 φⱼ(x) 朝 φⱼ(x₀) 靠近一点点。
循环。重复 3–4 步几百次。当 x 的特征逼近靶子，这张被优化出来的 x 就是反演结果。
特征重建损失（+ 正则）
$$\ell \;=\; \lVert\,\phi_j(x) - \phi_j(x_0)\,\rVert^2 \;+\; \lambda \cdot R(x)$$
前一项：在特征空间对齐靶子；后一项 R(x)：正则（如总变差 TV），让结果别太花、太噪。
02 — 最易错的一点

优化的是图像，不是网络

这是整个方法最反直觉、也最该记牢的地方。我们平时说「训练」，改的是网络的权重；但在特征反演里，网络（VGG）一个参数都不动，梯度下降修改的对象是那张图的像素本身。

The thing being optimized.

梯度回传，最终落在 像素 上，而不是 权重 上。

VGG 只是一把固定的「尺子」，负责把图像翻译成特征好做比较。真正被一点点雕出来的，是输入端那张图。所以反演没有「训练」出任何东西——它是对单张图像做的一次优化。

不变

VGG 的权重（冻结）
目标特征 φⱼ(x₀)（只算一次）

被优化

图像 x 的每一个像素
← 梯度下降反复修改

03 — 它揭示了什么

层越深，越抽象

在不同层做反演，还原出的图差别很大——这正是 M&V 想看的「不同层保留了什么信息」：

这个现象本身就是感知损失能成立的根基：高层特征「记得内容、忘了像素」，所以拿它当内容损失，能要求「内容像」却不强求「像素一模一样」——这正是后来 Gatys、Johnson 想要的性质。

04 — Gatys 怎么用它

同一个循环，加一个风格损失

Gatys 的风格迁移，沿用的就是上面 M&V 那个「优化图像」的循环——图像 → VGG → 算损失 → 梯度改像素 → 再来一遍。唯一的改动是：损失从一个变成两个，靶子从一张图变成两张（一张定内容、一张定风格）。

内容损失就是 M&V 的特征重建损失（保证「画的还是那张照片」）；风格损失比较 Gram 矩阵（保证「笔触、色调像那幅名画」）。两者加权求和，把画布 x 同时往「内容 A + 风格 B」拉。代价和 M&V 完全一样——每张图都要现场迭代，慢。

05 — Dosovitskiy & Brox 怎么提速

把「慢循环」换成一个网络

M&V 和 Gatys 的痛点是同一个：每张图都要从头跑几百步优化。Dosovitskiy & Brox 的想法很直接——训练一个前馈网络，让它一步就把特征还原成图像，不再逐图优化。

那这个网络到底拿什么当训练目标？原文的目标函数(公式 3)很朴素：

D&B 训练目标 · 逐像素 L2（原文公式 3）
$$\hat{w} \;=\; \arg\min_{w}\ \sum_i \big\lVert\,x_i - f(\phi_i,\, w)\,\big\rVert^2$$
网络输出 f(φᵢ) 与原图 xᵢ 的逐像素平方差。给定大量「图像 ↔ 特征」对 {xᵢ, φᵢ}，只训练解码网络 f 的权重 w（编码端/特征提取始终冻结，这也是它和自编码器的关键区别）。

注意它和 M&V 的损失比的东西不一样：M&V 在特征空间比 $\lVert\phi(x)-\phi(x_0)\rVert^2$，D&B 却在像素空间比。为什么 D&B 能、也愿意比像素？三层原因，从浅到深：

它手里有「标准答案图」。M&V 反演时没有目标图、只有目标特征，只能在特征空间比；而 D&B 是监督训练，先拿图 xᵢ 算出特征 φᵢ，天然就有了 (特征 → 原图) 的配对。原图既然在手边，最直接的监督就是「让网络输出尽量等于这张已知原图」。
逐像素 MSE 的最优解恰好是「期望原像」。这是原文特意点明的：最小化该损失，其最优解是条件期望 $\hat{f}(\phi_0) = \mathbb{E}[\,x \mid \phi=\phi_0\,]$，也就是所有可能产生这个特征的自然图像的(加权)平均。他们要回答「这个特征平均长什么样」，而逐像素 MSE 的数学最优解正好就是这张平均图——所以逐像素不是随便选的，是和研究目标精确对应的。
模糊本身变得「有信息」。因为最优解是平均图，重建的模糊程度恰好对应表示的不变性：层越高越不变(越多不同的图映射到同一特征)，平均出来就越糊。对「看每层保留了什么」这个目的来说，糊本身就是结论。

顺带一个对比能量化提速的来源：M&V 测试时要对特征求梯度做优化，慢(GPU 上约 6 秒/张)，且无法用于 LBP 这类不可微的特征；D&B 只在训练时贵，重建时一次前向约 5 毫秒/张——这就是约 1000× 提速的来源。

Why it comes out blurry.

逐像素 MSE 的最优解是 期望原像 $\mathbb{E}[x \mid \phi]$——「所有可能原图的平均」，所以重建天生偏糊。

这正是逐像素损失抓不住感知相似性的体现：它奖励「平均上对」，而非「看起来像」。D&B 把「慢优化 → 快网络」这半步做完了，但损失停在逐像素；Johnson 接手时几乎照搬了前馈结构，关键只改一处——把逐像素损失换成感知损失。于是「为什么糊」和「为什么 Johnson 要改」是同一个答案。

速度换来了：测试时只需一次前向。但有个关键点——这个解码网络训练时用的是逐像素损失。「前馈替代慢优化」的点子有了，损失却没升级。这一环，正是 Johnson 接下来要改的。

06 — 集大成

三块拼起来，就是 Johnson

把这条线连起来就清楚了：特征重建损失 $\lVert\phi_j(x)-\phi_j(x_0)\rVert^2$ 从头到尾没变过——在 Johnson 的引用链里，它直接来自 M&V [7]（用于反演 CNN 特征、理解网络层保留了什么信息）；Gatys 借它当内容损失、再加风格损失；D&B 想到用前馈网络替代慢优化。它们和 Johnson 的根本分野，都落在一句话上：梯度下降在改谁。

Same loss, different target.

同一套损失，优化图像 还是 优化网络，分出两条路。

M&V · Gatys

优化图像像素
每张图都要现场迭代 → 慢
用途：理解网络 / 风格迁移

D&B · Johnson

优化网络权重
训练一次，推理只跑一遍 → 快
用途：一步还原 / 实时生成

前馈这半边，D&B 先做了，但它训练用逐像素损失；Johnson 的关键一步，是把损失换成感知损失（内容来自 M&V、风格来自 Gatys）——于是前馈的「快」配上感知损失的「质」，集大成。论文那句话，本质就是在给这套损失上户口：源头在 Mahendran & Vedaldi（更早可追到 HOG、LBP 等手工特征的反演）。
